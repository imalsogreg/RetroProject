Record filtering

The contents of most (if not all) MWL file types consist of a list of records. For example a waveform file contains as many records as spikes were recorded, and every records consists of a timestamp field and a waveform field. Several import functions supoort selection (i.e. filtering) of records in the file. This filtering can be done on record index, on spike index or on time (units can be timestamps or seconds). The indices used for filtering can be either specified in block format (i.e. start index and end index) or in array format (i.e. a vector of indices). In case of filtering on time, only block format can be used. The [option string] and [index] parameters are used to set the filtering behavior. The option string is a list of characters that set the following options:

filter type:
  • (n)one – no filtering, get all records
  • spike (i)ndex – filter on spike index (only valid for tt2mat and fv2mat)
  • (r)ecord index – filter on record index (not valid for eeg2mat)
  • (t)imestamps – filter on time, using timestamps
  • (s)econds – filter on time using seconds
indexing type:
  • (a)rray – indices are specified as a vector of indices (e.g. [1 4 7 9 3 1])
  • (b)lock – indices or time boundaries are specifed as one or more blocks (i.e. 2- element vectors specifying start and end) (e.g. [ 0 100 ])
relative indexing:
  • (|) - indices / times are specified relative to the first index / time of first record in the file
  • (>) - indices / times are specified relative to the first index / time specified in the [index] parameter.
verbose level:
  • (0) – only error messages are displayed
  • (1) – error and warning messages are shown
  • (2) – error, warning and some info messages are shown
  • (3) – detailed information
data returned:
  • (+) - return all data
  • (-) - return timestamps and id only
See Examples for how these parameters can be used to import only part of the data in a file
